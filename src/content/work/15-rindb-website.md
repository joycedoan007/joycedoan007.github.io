<div style="margin: 0 0 20px;">
  <div class="overline" style="color: var(--text-accent); margin-bottom: 8px;">01 — Context</div>
  <h2 style="font: 600 22px/1.2 'Poppins', sans-serif; letter-spacing: -0.02em; color: var(--text-bright); margin: 0;">The Challenge</h2>
</div>

RinDB is a lightweight, embeddable key-value database built for developers who need fast, low-overhead storage without the complexity of a full database engine. The product is technically solid — the problem was communicating that to the right audience in a way they'd actually trust.

Developer tool marketing sits in a hard place. Enterprise database sites oversell and lose credibility with engineers immediately. Open-source project pages often have zero design — technically credible, but hard to evaluate quickly. RinDB needed to sit in between: technically serious, visually clean, and fast to parse.

<div style="background: var(--bg-surface); border-left: 2px solid var(--primary-500); border-radius: 0 var(--radius-lg) var(--radius-lg) 0; padding: 20px 24px; margin: 28px 0 56px;">
  <div class="overline" style="color: var(--text-accent); margin-bottom: 8px;">My Role</div>
  <p style="font: 400 15px/26px 'Poppins', sans-serif; color: var(--text-primary); margin: 0;">I owned the full design process end-to-end — empathy mapping, information architecture, sitemap, wireframes, design system, Figma file management, and high-fidelity UI. This was a solo design project with no handoff to another designer.</p>
</div>

<div style="margin: 0 0 20px;">
  <div class="overline" style="color: var(--text-accent); margin-bottom: 8px;">02 — Discovery</div>
  <h2 style="font: 600 22px/1.2 'Poppins', sans-serif; letter-spacing: -0.02em; color: var(--text-bright); margin: 0;">Research & Key Insights</h2>
</div>

Before touching pixels, I mapped out who the users are and what they actually need from a developer tool landing page. Four patterns shaped every decision that followed.

<div style="display: grid; grid-template-columns: 1fr 1fr; gap: 12px; margin: 24px 0 56px;">
  <div style="background: var(--bg-surface); border: 1px solid var(--border-subtle); border-radius: var(--radius-xl); padding: 24px;">
    <div style="font: 600 11px/16px 'JetBrains Mono', monospace; color: var(--text-accent); margin-bottom: 10px;">01</div>
    <p style="font: 600 14px/20px 'Poppins', sans-serif; color: var(--text-bright); margin: 0 0 8px;">Developers evaluate, they don't browse</p>
    <p style="font: 400 13px/22px 'Poppins', sans-serif; color: var(--text-secondary); margin: 0;">Engineers arrive with specific questions: Is this production-ready? How does it compare to SQLite or LevelDB? Can I try it in 5 minutes? The site needs to answer those — fast.</p>
  </div>
  <div style="background: var(--bg-surface); border: 1px solid var(--border-subtle); border-radius: var(--radius-xl); padding: 24px;">
    <div style="font: 600 11px/16px 'JetBrains Mono', monospace; color: var(--text-accent); margin-bottom: 10px;">02</div>
    <p style="font: 600 14px/20px 'Poppins', sans-serif; color: var(--text-bright); margin: 0 0 8px;">Marketing language kills credibility</p>
    <p style="font: 400 13px/22px 'Poppins', sans-serif; color: var(--text-secondary); margin: 0;">Developers are highly allergic to buzzwords and vague value props. Specificity builds trust — concrete numbers, real use cases, and direct language signal that the product is real.</p>
  </div>
  <div style="background: var(--bg-surface); border: 1px solid var(--border-subtle); border-radius: var(--radius-xl); padding: 24px;">
    <div style="font: 600 11px/16px 'JetBrains Mono', monospace; color: var(--text-accent); margin-bottom: 10px;">03</div>
    <p style="font: 600 14px/20px 'Poppins', sans-serif; color: var(--text-bright); margin: 0 0 8px;">Time-to-try is the real conversion metric</p>
    <p style="font: 400 13px/22px 'Poppins', sans-serif; color: var(--text-secondary); margin: 0;">A developer who runs a CLI command has already committed. Getting them to that moment — rather than clicking a "Contact Sales" CTA — is what actually converts in this space.</p>
  </div>
  <div style="background: var(--bg-surface); border: 1px solid var(--border-subtle); border-radius: var(--radius-xl); padding: 24px;">
    <div style="font: 600 11px/16px 'JetBrains Mono', monospace; color: var(--text-accent); margin-bottom: 10px;">04</div>
    <p style="font: 600 14px/20px 'Poppins', sans-serif; color: var(--text-bright); margin: 0 0 8px;">Visual language signals culture</p>
    <p style="font: 400 13px/22px 'Poppins', sans-serif; color: var(--text-secondary); margin: 0;">Dark theme, monospace typography, and code-adjacent aesthetics signal that this is a tool built by engineers for engineers — before the user reads a single word.</p>
  </div>
</div>

<div style="margin: 0 0 20px;">
  <div class="overline" style="color: var(--text-accent); margin-bottom: 8px;">03 — Design</div>
  <h2 style="font: 600 22px/1.2 'Poppins', sans-serif; letter-spacing: -0.02em; color: var(--text-bright); margin: 0;">Key Design Decisions</h2>
</div>

<div style="display: flex; flex-direction: column; gap: 10px; margin: 24px 0 56px;">
  <div style="border-left: 2px solid var(--primary-500); background: var(--bg-surface); border-radius: 0 var(--radius-xl) var(--radius-xl) 0; padding: 24px 28px;">
    <div class="overline" style="color: var(--text-accent); margin-bottom: 8px;">Decision 01</div>
    <p style="font: 600 15px/1.4 'Poppins', sans-serif; color: var(--text-bright); margin: 0 0 10px;">CLI example above the fold</p>
    <p style="font: 400 14px/24px 'Poppins', sans-serif; color: var(--text-primary); margin: 0;">Most developer tool sites bury the actual usage behind marketing copy. I pushed the CLI example high — close to the hero — so developers could see exactly what using RinDB looks like before committing to reading further. Showing the interface is the strongest trust signal available.</p>
  </div>
  <div style="border-left: 2px solid var(--primary-500); background: var(--bg-surface); border-radius: 0 var(--radius-xl) var(--radius-xl) 0; padding: 24px 28px;">
    <div class="overline" style="color: var(--text-accent); margin-bottom: 8px;">Decision 02</div>
    <p style="font: 600 15px/1.4 'Poppins', sans-serif; color: var(--text-bright); margin: 0 0 10px;">Dark palette with red-orange accent</p>
    <p style="font: 400 14px/24px 'Poppins', sans-serif; color: var(--text-primary); margin: 0;">The dark background (<code style="font: 500 12px/18px 'JetBrains Mono', monospace; color: var(--text-accent);">#0D0D0D</code>) with red-orange accents (<code style="font: 500 12px/18px 'JetBrains Mono', monospace; color: var(--text-accent);">#C0392B → #E74C3C</code>) was a deliberate cultural signal. It reads as low-level, performance-focused, and engineer-native — not a SaaS dashboard. The contrast also ensures code blocks stay highly readable.</p>
  </div>
  <div style="border-left: 2px solid var(--primary-500); background: var(--bg-surface); border-radius: 0 var(--radius-xl) var(--radius-xl) 0; padding: 24px 28px;">
    <div class="overline" style="color: var(--text-accent); margin-bottom: 8px;">Decision 03</div>
    <p style="font: 600 15px/1.4 'Poppins', sans-serif; color: var(--text-bright); margin: 0 0 10px;">Single-page flat architecture</p>
    <p style="font: 400 14px/24px 'Poppins', sans-serif; color: var(--text-primary); margin: 0;">A multi-page site would have fragmented the evaluation journey. A single scroll keeps the developer in context as they move through value prop → features → architecture → try it. Each section answers the next likely question, reducing the need to navigate away and come back.</p>
  </div>
</div>

  <div class="overline" style="color: var(--text-accent); margin-bottom: 8px;">06 — Outcome</div>
  <h2 style="font: 600 22px/1.2 'Poppins', sans-serif; letter-spacing: -0.02em; color: var(--text-bright); margin: 0;">Result</h2>

RinDB got a marketing site that matches the quality of the product — technically credible, fast to scan, and immediately actionable. The visual language communicates engineer culture before a word is read. The page flow gets a developer from "what is this" to "let me try it" without friction.

<div style="background: var(--bg-surface); border: 1px solid var(--border-default); border-radius: var(--radius-xl); padding: 28px 32px; margin-top: 28px;">
  <div class="overline" style="color: var(--text-accent); margin-bottom: 12px;">Reflection</div>
  <p style="font: 400 15px/26px 'Poppins', sans-serif; color: var(--text-primary); margin: 0;">Designing for a developer audience is humbling — they read design choices critically, not just aesthetically. The biggest thing I learned here is that restraint is a feature. Every element I considered adding, I asked: does this help a developer make a decision faster, or does it slow them down? If the answer wasn't clearly "faster," it didn't make the cut.</p>
</div>
